## Get Global Feed Events

```http
GET /feed/global
```

Retrieve the global activity feed showing recent events from all users across TrackGeek. This endpoint does not require authentication.

### Query Parameters

<ParamField query="cursor" type="string">
  Pagination cursor for getting the next page of results (optional)
</ParamField>

<ParamField query="itemsPerPage" type="number">
  Number of items to return per page (min: 1, max: 50, optional)
</ParamField>

### Example Request

```bash
curl -X GET 'https://api.trackgeek.com/feed/global?itemsPerPage=20'
```

## Get User Feed Events

<Info>
  This endpoint requires authentication. Retrieves a personalized activity feed showing events from users you follow.
</Info>

```http
GET /feed/user
```

Retrieve a personalized feed of activity from users you follow, showing their recent reviews, watches, favorites, and other activities.

### Query Parameters

<ParamField query="cursor" type="string">
  Pagination cursor for getting the next page of results (optional)
</ParamField>

<ParamField query="itemsPerPage" type="number">
  Number of items to return per page (min: 1, max: 50, optional)
</ParamField>

### Authentication

This endpoint requires a valid session token in the request headers.

### Example Request

```bash
curl -X GET 'https://api.trackgeek.com/feed/user?itemsPerPage=20' \
  -H 'Authorization: Bearer YOUR_SESSION_TOKEN'
```

## Feed Event Model

<ResponseField name="id" type="string">
  Unique identifier (UUID)
</ResponseField>

<ResponseField name="type" type="FeedEventType">
  Type of activity event
</ResponseField>

<ResponseField name="userId" type="string">
  ID of the user who performed the activity
</ResponseField>

<ResponseField name="metadata" type="object">
  Additional data specific to the event type (JSON)
</ResponseField>

<ResponseField name="createdAt" type="string">
  ISO 8601 timestamp of when the event occurred
</ResponseField>

<ResponseField name="user" type="object">
  User information including name, username, and avatar
</ResponseField>

## FeedEventType Enum

The `FeedEventType` enum specifies what type of activity occurred:

- `NewFollower` - User gained a new follower
- `NewFavorite` - User added a new favorite anime, manga, TV show, movie, game, or book
- `NewList` - User created a new list
- `NewListItem` - User added an item to a list
- `NewReview` - User posted a new review
- `NewWatch` - User started watching or completed anime, TV show, or movie
- `NewProgress` - User updated their progress on content
