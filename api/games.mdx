## Search Games

<Note>
  This endpoint does not require authentication.
</Note>

Search for games by title or keywords.

<CodeGroup>

```bash cURL
curl -X GET "https://api.trackgeek.com/game/search?query=witcher" \
  -H "Content-Type: application/json"
```

```javascript JavaScript
const response = await fetch('https://api.trackgeek.com/game/search?query=witcher', {
  method: 'GET',
  headers: {
    'Content-Type': 'application/json'
  }
});

const data = await response.json();
```

```python Python
import requests

response = requests.get(
    'https://api.trackgeek.com/game/search',
    params={'query': 'witcher'},
    headers={'Content-Type': 'application/json'}
)

data = response.json()
```

</CodeGroup>

### Query Parameters

<ParamField query="query" type="string" required>
  Search query string. Must be at least 3 characters long.
</ParamField>

      IGDB (Internet Game Database) ID
    </ResponseField>
    
    <ResponseField name="name" type="string">
      Game name
    </ResponseField>
    
    <ResponseField name="slug" type="string">
      URL-friendly slug
    </ResponseField>
    
    <ResponseField name="coverUrl" type="string">
      Cover image URL
    </ResponseField>
    
    <ResponseField name="summary" type="string">
      Game summary/description
    </ResponseField>
    
    <ResponseField name="firstReleaseDate" type="string">
      First release date (ISO 8601)
    </ResponseField>
    
    <ResponseField name="genres" type="array">
      Array of genre objects
    </ResponseField>
    
    <ResponseField name="platforms" type="array">
      Array of platform objects
    </ResponseField>
    
    <ResponseField name="rating" type="number">
      User rating (0-100)
    </ResponseField>
  </Expandable>
</ResponseField>

## Get Game Details

<Note>
  This endpoint does not require authentication.
</Note>

Retrieve detailed information about a specific game by its ID.

<CodeGroup>

```bash cURL
curl -X GET "https://api.trackgeek.com/game/details/1942" \
  -H "Content-Type: application/json"
```

```javascript JavaScript
const gameId = 1942;
const response = await fetch(`https://api.trackgeek.com/game/details/${gameId}`, {
  method: 'GET',
  headers: {
    'Content-Type': 'application/json'
  }
});

const data = await response.json();
```

```python Python
import requests

game_id = 1942
response = requests.get(
    f'https://api.trackgeek.com/game/details/{game_id}',
    headers={'Content-Type': 'application/json'}
)

data = response.json()
```

</CodeGroup>

## Refresh Game

<Note>
  This endpoint requires authentication. Include your API token in the Authorization header.
</Note>

Refresh game data from IGDB to get the latest information.

<CodeGroup>

```bash cURL
curl -X POST "https://api.trackgeek.com/game/refresh" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "id": 1942
  }'
```

```javascript JavaScript
const response = await fetch('https://api.trackgeek.com/game/refresh', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer YOUR_TOKEN',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    id: 1942
  })
});
```

```python Python
import requests

response = requests.post(
    'https://api.trackgeek.com/game/refresh',
    headers={
        'Authorization': 'Bearer YOUR_TOKEN',
        'Content-Type': 'application/json'
    },
    json={'id': 1942}
)
```

</CodeGroup>

### Request Body

<ParamField body="id" type="integer" required>
  The IGDB ID of the game to refresh. Must be a valid integer.
</ParamField>

### Response

Returns HTTP 200 on successful refresh with no body content.

<ResponseExample>

```json Success Response
{}
```

</ResponseExample>
