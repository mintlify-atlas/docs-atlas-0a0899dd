---
title: 'Reviews'
description: 'Create, update, and manage reviews for all media types'
---

<Note>
Reviews allow users to rate and critique media with detailed breakdowns. Each media type has specific rating categories.
</Note>

## Review Field Validations

All review endpoints enforce these validations:

- **Rating fields**: Decimal values between 0 and 10
- **summary**: Max 250 characters
- **notes**: Max 1000 characters
- **pros/cons** (anime only): Max 500 characters each
- **story/characters** (manga only): Max 500 characters each

## Anime Reviews

### Create Anime Review

<Tabs>
  <Tab title="Request">
    ```bash POST /api/anime/review
    curl -X POST https://api.trackgeek.com/anime/review \
      -H "Authorization: Bearer YOUR_TOKEN" \
      -H "Content-Type: application/json" \
      -d '{
        "animeId": "anime-uuid-here",
        "overall": 8.5,
        "story": 9.0,
        "characters": 8.0,
        "animation": 9.5,
        "sound": 8.5,
        "enjoyment": 9.0,
        "summary": "An excellent anime with great character development",
        "pros": "Beautiful animation, compelling story",
        "cons": "Pacing issues in middle episodes",
        "notes": "Additional thoughts and observations",
        "recommended": true
      }'
    ```
  </Tab>
  <Tab title="Response">
    ```json 201 Created
    {
      "id": "review-uuid",
      "overall": 8.5,
      "story": 9.0,
      "characters": 8.0,
      "animation": 9.5,
      "sound": 8.5,
      "enjoyment": 9.0,
      "summary": "An excellent anime with great character development",
      "pros": "Beautiful animation, compelling story",
      "cons": "Pacing issues in middle episodes",
      "notes": "Additional thoughts and observations",
      "recommended": true,
      "userId": "user-uuid",
      "animeId": "anime-uuid",
      "createdAt": "2024-03-02T12:00:00Z",
      "updatedAt": "2024-03-02T12:00:00Z"
    }
    ```
  </Tab>
</Tabs>

<ParamField body="animeId" type="string" required>
  The UUID of the anime being reviewed
</ParamField>

<ParamField body="overall" type="number" required>
  Overall rating (0-10). Required field that represents the comprehensive score.
</ParamField>

<ParamField body="story" type="number">
  Story/plot rating (0-10)
</ParamField>

<ParamField body="characters" type="number">
  Character development rating (0-10)
</ParamField>

<ParamField body="animation" type="number">
  Animation quality rating (0-10)
</ParamField>

<ParamField body="sound" type="number">
  Sound/music rating (0-10)
</ParamField>

<ParamField body="enjoyment" type="number">
  Personal enjoyment rating (0-10)
</ParamField>

<ParamField body="summary" type="string">
  Brief review summary (max 250 characters)
</ParamField>

<ParamField body="pros" type="string">
  Positive aspects (max 500 characters)
</ParamField>

<ParamField body="cons" type="string">
  Negative aspects (max 500 characters)
</ParamField>

<ParamField body="notes" type="string">
  Additional notes (max 1000 characters)
</ParamField>

<ParamField body="recommended" type="boolean">
  Whether you recommend this anime
</ParamField>

### Get Anime Reviews

```bash GET /api/anime/review
curl "https://api.trackgeek.com/anime/review?animeId=anime-uuid-here&offset=0&limit=20"
```

<ParamField query="animeId" type="string">
  Filter reviews by anime UUID
</ParamField>

<ParamField query="userId" type="string">
  Filter reviews by user UUID
</ParamField>

<ParamField query="offset" type="number">
  Pagination offset (default: 0)
</ParamField>

<ParamField query="limit" type="number">
  Results per page (default: 20)
</ParamField>

### Get Anime Review by ID

```bash GET /api/anime/review/{reviewId}
curl https://api.trackgeek.com/anime/review/{reviewId}
```

<ParamField path="reviewId" type="string" required>
  The UUID of the review
</ParamField>

### Update Anime Review

```bash PATCH /api/anime/review/{reviewId}
curl -X PATCH https://api.trackgeek.com/anime/review/{reviewId} \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "overall": 9.0,
    "animation": 10.0
  }'
```

<Note>
Only the review author can update their review. All fields from create are available for update.
</Note>

### Delete Anime Review

```bash DELETE /api/anime/review/{reviewId}
curl -X DELETE https://api.trackgeek.com/anime/review/{reviewId} \
  -H "Authorization: Bearer YOUR_TOKEN"
```

<Warning>
Deleting a review is permanent and cannot be undone.
</Warning>

<ResponseField name="id" type="string">
  Review UUID
</ResponseField>

<ResponseField name="userId" type="string">
  Author's user UUID
</ResponseField>

<ResponseField name="createdAt" type="string">
  ISO 8601 timestamp of creation
</ResponseField>

<ResponseField name="updatedAt" type="string">
  ISO 8601 timestamp of last update
</ResponseField>

## Authentication

All POST, PATCH, and DELETE endpoints require authentication via Bearer token:

```bash
Authorization: Bearer YOUR_TOKEN
```

## Constraints

- Users can only have one review per media item (enforced by unique constraint on `userId` + media ID)
- Only the review author can update or delete their review
- All rating fields must be decimal values between 0 and 10
- The `overall` field is always required when creating a review
