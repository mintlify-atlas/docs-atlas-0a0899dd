# Quickstart Guide

This guide will walk you through setting up the TrackGeek API locally and making your first authenticated request.

## Prerequisites

Before you begin, ensure you have the following installed:

- **Node.js** (v18 or higher)
- **Docker** and Docker Compose
- **Git**
- **PostgreSQL** (via Docker)
- **Redis** (via Docker)

## Installation

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/TrackGeek/api.git
    cd api
    ```
  </Step>

  <Step title="Install dependencies">
    ```bash
    npm install
    ```
  </Step>

  <Step title="Set up environment variables">
    Copy the example environment file and configure it:

    ```bash
    cp .env.example .env
    ```

    Update the `.env` file with your configuration:

    ```bash .env
    # SERVER
    PORT=40287

    # DATABASE
    DATABASE_URL='postgresql://user:password@localhost:5432/trackgeek'

    # CACHE
    REDIS_URL='redis://localhost:6379'

    # AUTH
    BETTER_AUTH_URL='http://localhost:40287'
    BETTER_AUTH_SECRET='your-secret-key-here'
    WEB_URL='http://localhost:5173'

    # Social Providers (Optional - configure only what you need)
    GOOGLE_CLIENT_ID=''
    GOOGLE_CLIENT_SECRET=''
    DISCORD_CLIENT_ID=''
    DISCORD_CLIENT_SECRET=''
    GITHUB_CLIENT_ID=''
    GITHUB_CLIENT_SECRET=''

    # EMAIL
    RESEND_API_KEY=''
    RESEND_FROM=''

    # INTEGRATIONS
    IMGBB_API_KEY=''
    HARDCOVER_API_KEY=''
    TMDB_API_KEY=''
    IGDB_CLIENT_ID=''
    IGDB_CLIENT_SECRET=''
    ```

    <Note>
      Generate a secure random string for `BETTER_AUTH_SECRET` using: `openssl rand -base64 32`
    </Note>
  </Step>

  <Step title="Start Docker containers">
    Start PostgreSQL and Redis:

    ```bash
    docker compose up -d
    ```
  </Step>

  <Step title="Run database migrations">
    Set up the database schema:

    ```bash
    npm run prisma:migrate
    npm run prisma:generate
    ```
  </Step>

  <Step title="Start the development server">
    ```bash
    npm run dev
    ```

    You should see:

    ```
    Server is running on http://localhost:40287
    API documentation available at http://localhost:40287/docs
    ```
  </Step>
</Steps>

## Your First Request

Now that the API is running, let's make some requests!

### Search for Anime (Public Endpoint)

Some endpoints don't require authentication. Let's search for anime:

<CodeGroup>

```bash cURL
curl http://localhost:40287/anime/search?q=naruto
```

```javascript JavaScript
const response = await fetch('http://localhost:40287/anime/search?q=naruto');
const data = await response.json();
console.log(data.animes);
```

```python Python
import requests

response = requests.get('http://localhost:40287/anime/search', params={'q': 'naruto'})
data = response.json()
print(data['animes'])
```

</CodeGroup>

### Create an Account

To access protected endpoints, you need to authenticate. Let's create an account:

<CodeGroup>

```bash cURL
curl -X POST http://localhost:40287/auth/sign-up/email \
  -H "Content-Type: application/json" \
  -d '{
    "email": "user@example.com",
    "password": "securePassword123",
    "name": "John Doe"
  }'
```

```javascript JavaScript
const response = await fetch('http://localhost:40287/auth/sign-up/email', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
  },
  credentials: 'include',
  body: JSON.stringify({
    email: 'user@example.com',
    password: 'securePassword123',
    name: 'John Doe'
  })
});

const data = await response.json();
console.log(data);
```

```python Python
import requests

session = requests.Session()

response = session.post(
    'http://localhost:40287/auth/sign-up/email',
    json={
        'email': 'user@example.com',
        'password': 'securePassword123',
        'name': 'John Doe'
    }
)

data = response.json()
print(data)
```

</CodeGroup>

<Info>
  The sign-up endpoint automatically creates a user profile with a generated username. Session cookies are set automatically for subsequent requests.
</Info>

### Sign In

If you already have an account, sign in to get your session:

<CodeGroup>

```bash cURL
curl -X POST http://localhost:40287/auth/sign-in/email \
  -H "Content-Type: application/json" \
  -c cookies.txt \
  -d '{
    "email": "user@example.com",
    "password": "securePassword123"
  }'
```

```javascript JavaScript
const response = await fetch('http://localhost:40287/auth/sign-in/email', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
  },
  credentials: 'include',
  body: JSON.stringify({
    email: 'user@example.com',
    password: 'securePassword123'
  })
});

const data = await response.json();
console.log(data.user);
```

```python Python
import requests

session = requests.Session()

response = session.post(
    'http://localhost:40287/auth/sign-in/email',
    json={
        'email': 'user@example.com',
        'password': 'securePassword123'
    }
)

data = response.json()
print(data['user'])
```

</CodeGroup>

### Make an Authenticated Request

Now that you're authenticated, let's update your profile:

<CodeGroup>

```bash cURL
curl -X PATCH http://localhost:40287/profile \
  -b cookies.txt \
  -H "Content-Type: application/json" \
  -d '{
    "bio": "Anime enthusiast and gamer",
    "timezone": "America/New_York"
  }'
```

```javascript JavaScript
const response = await fetch('http://localhost:40287/profile', {
  method: 'PATCH',
  headers: {
    'Content-Type': 'application/json',
  },
  credentials: 'include',
  body: JSON.stringify({
    bio: 'Anime enthusiast and gamer',
    timezone: 'America/New_York'
  })
});

if (response.ok) {
  console.log('Profile updated successfully!');
}
```

```python Python
# Using the same session from sign-in
response = session.patch(
    'http://localhost:40287/profile',
    json={
        'bio': 'Anime enthusiast and gamer',
        'timezone': 'America/New_York'
    }
)

if response.status_code == 200:
    print('Profile updated successfully!')
```

</CodeGroup>

Fetch detailed information about a specific anime:

<CodeGroup>

```bash cURL

```

```javascript JavaScript
const response = await fetch('http://localhost:40287/anime/details/1535');

```

```python Python
response = requests.get('http://localhost:40287/anime/details/1535')
```

</CodeGroup>

## Response Format

All successful responses return JSON with a consistent structure:

```json Success Response
{
  "anime": {
    "id": 1535,
    "title": "Death Note",
    "synopsis": "...",
    "status": "finished_airing",
    "episodes": 37
  }
}
```

Errors follow a standardized format:

```json Error Response
{
  "statusCode": 401,
  "message": "Unauthorized",
  "error": "Authentication required"
}
```

## Interactive API Documentation

TrackGeek provides interactive API documentation powered by Swagger/Scalar. Visit the docs to explore all endpoints:

```
http://localhost:40287/docs
```

<Info>
  The interactive documentation is only available in development mode (`NODE_ENV=development`).
</Info>

## Common Response Codes

| Code | Meaning | Description |
|------|---------|-------------|
| 200 | OK | Request succeeded |
| 201 | Created | Resource created successfully |
| 204 | No Content | Request succeeded with no response body |
| 400 | Bad Request | Invalid request parameters |
| 401 | Unauthorized | Authentication required |
| 403 | Forbidden | Insufficient permissions |
| 404 | Not Found | Resource not found |
| 422 | Unprocessable Entity | Validation failed |
| 429 | Too Many Requests | Rate limit exceeded |
| 500 | Internal Server Error | Server error |
